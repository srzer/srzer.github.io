8月 28, 2018 725

但是一个人失败也就罢了。。要是失败后还没有教训就是SB了。。–WJMZBMR

Problem

  • 聪明的Amyzhi
  • 石子游戏
  • 末日传说

Result

  • T1 60
  • T2 0
  • T3 0

Experience

扫一眼题目，发现第一题像扩展欧几里得，第二题像gcd线段树(瞎了眼了)，第三题一定是个动规，然后p都没想就开始写第一题的暴力。

吭哧吭哧很快就写好了，预计得分40。接着发现扩展欧几里得好像用不到，顶多在最后两个数字选择时候用一下，但是我竟然真的这样干了！！！

写完第二个完全没用的程序后，开始记忆化搜索，发现递推更好写，就写递推，处理了一会边界也写好了，预计60。

打完dp，没办法优化了，但是第一题看起来是个水题，拿不到100不行啊，所以只有打表找规律了。发现确实有规律，我觉得自己已经发现标算了！

接着发现我这个规律是片面的！但又发现了新的规律，只是不好处理。终于处理完了但样例还是过不了。现在已经将近两个半小时了..开始绝望。

于是迅速打起了第二题的暴力，发现不好写？？？调参数从100调到20，才把样例输对了，已经准备好TLE20了，然后斟酌一下发现第三题的暴力不一定有第一题的满分好写，所以继续找规律..

找了半天，好像又发现了，打了结果样例不过。时间到了，我绝望的交上了程序。

Lesson

  • 没参悟好数论就开始瞎用，忽视数据特殊性，无视最基础的枚举。

  • OJ上的专题做时间长了，都是不用分析算法的题目，考试的时候很难想到算法了..

  • 前段时间代码精准度还行，现在又下降，和Tsinsen上老是WA了再交有关..

  • 考试时候太急了。后面两题每道分析的时间都不足半小时，就会很焦虑..

综上，我决定去POJ做一段时间的题..

实战比赛 赛后总结
newer 聪明的Amyzhi
older 组合数学
