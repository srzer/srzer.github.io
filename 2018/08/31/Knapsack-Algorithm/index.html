背包问题

Aug 31, 2018

01背包问题

这是最基础的背包问题，特点是：每种物品仅有一件，可以选择放或不放。

Solution

对于每个物品都有选与不选两种选择，用子问题定义状态：dp[i][v]表示前i个物品放入一个容量为v的背包可以获得的最大价值，则其状态转移方程为：dp[i][v]=max(dp[i-1][v-w[i]]+c[i],dp[i-1][v])

为了省空间，可以通过改变枚举顺序for v from 1 to Maxn –> for v from Maxn to 1，直接覆盖i-1的状态，从而省去第一维。

Example

多米诺骨牌

完全背包问题

相对于01背包问题，每种物品都有无限件可用。

Solution

只需要在01背包的一维代码上再把枚举v的顺序颠倒回正即可，dp[v-w[i]]可能已经在之前更新过了，进行状态转移后dp[v]就可能存在k个i物品，从而满足完全背包要求。

Example

飞扬的小鸟

多重背包问题

相对于01背包问题，每件物品最多有n[i]件可用。

Solution

很显然，可以将每件物品拆分成n[i]件后进行01背包，复杂度为物品的总数。

还有优化方法吗？是的。

因为n[i]个物品的属性相同，所以可以将物体分成若干件物品，其中每件物品有一个系数，这件物品的费用和价值均是原来的费用和价值乘以这个系数。

定义k为满足n[i]>2\^k-1的最大整数，使这些系数分别为1,2,4,…,2\^(k-1),n[i]-2\^k+1，那么这k+1件物品就可以组合出0~n[i]个原物品，且每个数量只出现一次。将每一种物品的复杂度都从O(n)降到了O(logn)。

Example

庆功会

依赖背包问题

物品间存在某种依赖关系，若i依赖于j，则若选物品i，必选物品j。没有某个物品既依赖于其他物品，又被其他物品依赖，且一个物品只能依赖于至多一个物品。

Solution

按照一般思路，仅考虑一个主件和它的附件集合，对于一个拥有n个附件的主件共有2^n种策略，复杂度过高，只能适用于附件较少的情况。

因为所有的策略皆为互斥，则对于每个主件进行预处理，那么对于价值1~v，附件中都有一个最佳组合，做一个子01背包问题，可以将附件策略简化成v种。

这就是传说中的“泛化物品”思想..
